¿Qué es?

RWD (responsive web design), es una filosofía de diseño y desarrollo que busca adaptar la apariencia de las páginas web al dispositivo que se esté utilizando para visualizarla.

¿En qué consiste?

Redimensiona y coloca los elementos de la web de forma que se adapten al ancho de cada dispositivo, desde ordenadores de escritorio a tablets y móviles. Además, tiene en cuenta características concretas: tamaño de pantalla, resolución, potencia de CPU y capacidad de memoria.

Objetivo

Proporcionar a todos los usuarios los mismos contenidos frente a diferentes tipos de dispositivos, permitiendo una correcta visualización y una mejor experiencia de usuario. Brinda al usuario una imagen consistente.

Ventajas

  • Permite reducir el tiempo de desarrollo. Por ejemplo, necesitarías un sitio para teléfono móvil y otro para ordenador de escritorio.
  • Evita los contenidos duplicados.
  • Aumenta la viralidad de los contenidos ya que permite compartirlos de una forma mucho más rápida y natural.
  • Respeta las pautas de Google por su efectividad en el posicionamiento.
  • Google da prioridad en las búsquedas de sitios responsive sobre las webs unitarias.
  • Baja tasa de rebote, sucede cuando tu sitio web no cuenta con funciones adecuadas de interacción con el cliente, los usuarios abandonan tu página y Google baja tu página en el ranking de búsquedas.

Desventajas

  • Carga de página: no se diferencía a nivel de servidor la carga de elementos. Al hacer la web responsive se cargan exactamente los mismos recursos que para tu web de escritorio. Hay que evitar sobrecargar la página de elementos inútiles.
  • Efectos: Algunos de los efectos que usas en la web de escritorio no funcionarán en la web responsive. Ejemplo, el hover. Usa librerías que no carguen recursos innecesarios.

Puntos a tener en cuenta:

  • Evita efectos y elementos que puedan hacer lenta la carga.
  • Ten en cuenta los hábitos de navegación. Se necesitan soluciones rápidas y concretas.
  • Evita sobrecarga de información, usa textos cortos y botones para que tu usuario sepa exactamente qué debe hacer para navegarse a sí mismo.
  • Los tamaños de las fuentes, las imágenes, los menús y los elementos gráficos deben estar pensados para adaptarse a cualquier tamaño.
